======================
AYUDA - CÓMO USAR POSTE.AR
======================

POSTEAR UN MENSAJE
------------------
1. Escribí tu mensaje en el cuadro (máximo 576 caracteres)
2. Presioná [ postear ]
3. Tu mensaje aparecerá con tu marca temporal

TU MARCA
--------
Es tu identificador anónimo de esta sesión.
Formato: nana.12.yija.87 (sílabas + números)

Mientras no cierres la página o presiones [ reetiquetar ], 
todos tus posts llevarán la misma marca.

REETIQUETAR
-----------
Presioná [ reetiquetar ] para cambiar tu marca y empezar 
una nueva identidad anónima.

FORMATO DE TEXTO
----------------
*texto*           → texto en negrita
_texto_           → texto en cursiva
<nombre:url.com>  → crear enlace clickeable

Ejemplo:
Esto es *importante* y _urgente_.
Más info en <aquí:ejemplo.com>

BUSCAR MENSAJES
---------------
Escribí en el cuadro "Buscar..." lo que querés encontrar.

Ejemplos:
- Buscar "papeles" → muestra posts con esa palabra
- Buscar "papeles, Argentina" → muestra posts que tengan 
  AMBAS palabras (búsqueda con Y)
- Buscar "23" → encuentra en hora, fecha, texto, marca

La búsqueda NO distingue mayúsculas de minúsculas.

CONTENIDO OCULTO (próximamente)
--------------------------------
Usá ((secreto|clave123)) para cifrar parte del mensaje.
Solo quien tenga la clave "clave123" podrá leerlo.

NOTA: Esta función estará disponible próximamente.


======================
QUÉ ES POSTE.AR
======================

DEFINICIÓN
----------
Poste.ar es un sitio de comunicación anónima y efímera.
Un tablón digital donde publicar mensajes sin identidad 
permanente ni registro de usuarios.

FILOSOFÍA
---------
- Anonimato: No hay cuentas, emails ni contraseñas
- Efímero: Los mensajes se borran automáticamente
- Sin rastro: No guardamos IPs ni datos personales
- Abierto: El código es open source

PERSISTENCIA
------------
Todos los mensajes se eliminan automáticamente cada lunes 
a las 00:00 hs (hora Argentina).

El sitio muestra un contador indicando el % de capacidad 
de la base de datos.

PRIVACIDAD
----------
- No se registran direcciones IP
- No hay cookies de rastreo
- No hay análisis de comportamiento
- Todo el código es auditable

TECNOLOGÍA
----------
- Backend: Node.js + Express
- Base de datos: Turso (libSQL)
- Frontend: HTML, CSS, JavaScript vanilla
- Hosting: Render

UBICACIÓN
---------
Proyecto experimental desarrollado en Ciudadela, 
Buenos Aires, Argentina.

CÓDIGO FUENTE
-------------
Presioná [ download source ] para ver información 
del repositorio y la licencia del proyecto.

Versión: 1.0 Beta
